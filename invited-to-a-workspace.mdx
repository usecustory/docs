title: "Invited to a workspace"
description: "Join an existing Custory workspace cleanly so you land in the right context without creating duplicates or role confusion."

Use this guide when someone invited you to an existing Custory workspace and you want to get oriented quickly.

The goal is simple: join the correct workspace, understand your role, and start from the right journey instead of creating a parallel setup by mistake.

## What to do first

If you received an invitation:

1. accept the invite to the existing workspace
2. avoid creating a new workspace for the same team or product
3. confirm which journey or workflow you are expected to work in first

For most teams, duplicate workspaces create confusion faster than almost any other setup mistake.

## Understand the role you were given

Custory invitations usually assign either `Editor` or `Viewer`.

- `Editor` means you are expected to help maintain journeys, items, or workflows
- `Viewer` means you should follow the work without changing the source of truth

If you are not sure why you were given a role, ask the workspace owner before changing how you work.

See [Manage your team](/team-management) for the full role model.

## Get oriented inside the workspace

Once you join, start with three questions:

1. which journey matters most right now?
2. what decision is that journey helping the team make?
3. where does your part of the work fit into it?

If the team has not pointed you to a starting journey yet, use these pages first:

- [What is a customer journey?](/what-is-a-customer-journey)
- [How Custory works](/how-custory-works)
- [Items](/items)

## If you only need visibility

If you were invited mainly to follow progress:

- stay in viewer mode if possible
- use the journey to understand the customer story, not to create parallel notes
- configure [notifications](/notifications) only for the updates you truly need

This keeps the workspace readable while still giving you the context behind the work.

## If you are expected to contribute

If you were invited as an editor, the safest way to start is:

1. review the existing journey before editing it
2. inspect the linked [touchpoints](/touchpoints), [insights](/insights), and [opportunities](/opportunities)
3. ask which parts of the map are actively maintained
4. make changes close to the customer evidence, not as separate notes

That helps you join the team's system instead of accidentally creating drift.

## Common invitation mistakes

<AccordionGroup>
  <Accordion title="Creating a second workspace for the same team">
    If the team already has a workspace, join that one. Separate workspaces are for real access or product boundaries, not convenience.
  </Accordion>
  <Accordion title="Assuming viewer and editor roles behave the same">
    They do not. Viewer is for visibility. Editor is for active maintenance of the shared operating system.
  </Accordion>
  <Accordion title="Starting to edit before understanding the journey scope">
    Spend a few minutes understanding what flow the team mapped and what decision it supports before you add or rewrite anything.
  </Accordion>
</AccordionGroup>

## What a good join experience looks like

You should be able to answer:

- which workspace you belong in
- what your role allows you to do
- which journey to open first
- how your work connects back to the customer journey
