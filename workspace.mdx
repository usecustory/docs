A workspace is the top-level operating space for one team, one product, or one product area in Custory.

The workspace decision matters because it controls:

- Who can see the work
- Which journeys live together
- Which integrations are shared
- Which billing and usage settings apply
- Which AI memory and MCP access stay scoped together

## What belongs in one workspace

Keep work in the same workspace when the same team needs shared context.

Good reasons to stay in one workspace:

- The same people work across the same customer flows
- The journeys feed the same product decisions
- The integrations should be reused across the same work
- You want AI to work from one shared context pool

Create a separate workspace when you need clean separation:

- A different product
- A different client
- A different business unit
- A different access boundary
- A different billing owner

Do not create a new workspace just because you want a new journey. Use multiple journeys inside the same workspace first.

## Create a workspace

You can create a workspace from the workspace switcher.

Set the name, slug, and logo so the workspace is easy to recognize and share.

## Switch between workspaces

Use the workspace switcher in the sidebar to move between workspaces.

When you switch:

- Custory takes you into the selected workspace
- It tries to keep you near the same area of the product when that still makes sense
- It does not carry over journey-specific pages that belong to a different workspace

That behavior is useful for small teams that move between product areas during the day without wanting to manually rebuild context every time.

## Manage workspace details

Open [**Workspace settings**](https://app.usecustory.com/workspace-settings) to manage the workspace itself.

From there, owners can update:

- Workspace name
- Workspace logo
- Workspace slug
- Billing settings
- Usage settings
- Member access

## Leave or delete a workspace

These actions are not the same.

### Leave workspace

Use this when you personally should no longer have access, but the workspace should continue for the rest of the team.

Editors and viewers can leave a workspace from settings.

### Delete workspace

Use this only when the workspace and its associated data should be removed entirely.

Deletion is destructive. Treat it as an administrative action, not a cleanup shortcut.

When an owner deletes a workspace, Custory immediately cancels all paid plans and billing attached to that workspace. That includes the base subscription and any paid add-ons such as extra editor seats.

Cancellation is effective immediately. Custory should not continue charging for a workspace after it has been deleted.

## Good workspace habits

The healthiest workspaces usually have:

- A clear owner
- A small accountable editor group
- A narrow set of active journeys
- Only relevant integrations connected
- A review rhythm that keeps the map current
