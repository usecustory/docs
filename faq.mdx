## Do I need to map every customer journey?

No. Start with one journey that matters now.

Most teams get better results from one maintained journey than from several broad maps that nobody updates.

## Should I start from a blank journey?

Not always.

Custory supports blank journeys, templates, and AI-powered imports from connected sources such as GitHub, Figma, and Notion. Use the starting path that matches how much source material you already have.

## How detailed should a journey be?

Detailed enough to support decisions.

If a step is too broad, the team cannot act on it. If it is too detailed, the map becomes hard to maintain.

## What is the difference between a journey and an item?

A journey is the overall flow.

An item is a structured piece of context attached to that flow, such as an insight, opportunity, solution, metric, or touchpoint.

## Can we use Custory AI without leaving the app?

Yes.

Custory AI is built into journeys, personas, and automation workflows. It can also work with connected integrations and workspace memory.

## When should we use MCP?

Use MCP when your team wants external AI clients such as Cursor or Claude to work directly with Custory workspace context.

Use the in-product assistant when the work belongs naturally inside the Custory UI.

## Can Slack and Discord do more than receive notifications?

Yes.

They can receive automation messages and workspace updates, and they can also support AI collaboration and direct-message notifications when personal accounts are linked.

## Can Custory create work in our delivery tools?

Yes.

Custory can create and track follow-up work in tools such as GitHub, Jira, and Linear, and can keep external task titles and statuses more current inside the journey.

## What kinds of automations can we build?

You can build scheduled and event-based automations.

Examples include weekly summaries, GitHub PR follow-up, analytics checks from PostHog or Stripe, and workflows that create tasks or send Slack or Discord messages.

## How should we set up notifications?

Start with in-app notifications for everything.

Then add email, Slack DM, or Discord DM only for the updates that need faster attention, such as mentions, assignments, automation failures, or integration issues.
