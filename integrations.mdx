Custory integrations are designed to keep customer context connected to the places where your team already talks, builds, measures, and decides.

## Supported integration groups

Custory currently supports these groups:

- Product management: Jira, Linear
- Knowledge base: Notion, Google Drive
- Analytics and revenue: PostHog, Stripe
- Communication: Slack, Discord
- Developer tools: GitHub, MCP server
- Customer feedback: Intercom
- Collaborative editors: Figma, Miro

## Communication

### Slack

Use Slack to:

- Send journey updates to channels
- Deliver team-facing automation messages
- Link personal Slack accounts for direct message notifications
- Use Slack as an AI collaboration surface in relevant workflows

### Discord

Use Discord to:

- Share updates to channels and threads
- Deliver automation messages
- Link personal Discord accounts for direct message notifications
- Work from community and feedback-heavy conversations
- Use Discord threads with Custory AI

## Delivery and product work

### GitHub

Use GitHub to:

- Link issues and pull requests back to journey context
- Keep implementation progress visible inside Custory
- Trigger automations from PR events
- Feed AI and MCP workflows with real delivery context

### Jira and Linear

Use Jira and Linear to:

- Create follow-up work directly from Custory
- Keep task status and titles in sync
- Preserve the customer reason behind each task

## Knowledge and design

### Notion and Google Drive

Use these integrations when research, notes, or documentation already exist elsewhere and should inform journey work or AI prompts.

### Figma and Miro

Use these integrations when visual product context, flows, or collaborative boards should stay connected to the journey.

## Product and revenue signals

### PostHog

Use PostHog in automations to pull product usage snapshots such as event movement and top events.

### Stripe

Use Stripe in automations to pull subscription, payment failure, and refund signals that can reveal churn or billing friction.

## Customer feedback

### Intercom

Use Intercom when support and customer conversation history should stay connected to journey context for faster triage and pattern detection.

## Integration management

Open **Manage Integrations** inside a workspace to:

- Connect or disconnect tools
- Reauthorize expired integrations
- Review integration health
- Add manual credentials where required
- Request an integration that is not yet supported

## How to choose what to connect first

Start with the smallest set that closes your biggest context gaps.

For many teams that means:

1. Slack or Discord
2. GitHub, Jira, or Linear
3. Notion or Google Drive
4. PostHog or Stripe
