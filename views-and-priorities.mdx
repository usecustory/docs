Custory gives you several ways to work in the same journey.

Each view is built for a different job. You should switch views based on the question the team is trying to answer, not based on personal preference alone.

## When to use each view

<CardGroup cols={3}>
  <Card title="Journey view" icon="route" href="/journey-view">
    Best for understanding the flow, reviewing customer context, and telling the story of the experience.
  </Card>

  <Card title="Table view" icon="table" href="/table-view">
    Best for sorting, filtering, auditing, and cleaning up larger journeys.
  </Card>

  <Card title="Impact vs effort view" icon="chart-line" href="/impact-vs-effort-view">
    Best for deciding which opportunities and solutions should move first.
  </Card>
</CardGroup>

## Shared controls

These views all work on the same underlying journey and share the same working context:

- Search
- Filters
- Item properties
- Linked tasks
- Comments
- AI context

That means you can move from narrative review to cleanup to prioritization without recreating the work.
