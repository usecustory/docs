Custory lets you work with the same journey in different modes, depending on whether the team needs story, operations, or prioritization.

## Journey view

Use journey view when you need sequence and narrative.

Best for:

- Reviewing the full customer flow
- Running workshops
- Explaining the experience to teammates
- Seeing where friction repeats across stages

## Grid view

Use grid view when you want a more operational scan across journey content.

Best for:

- Comparing grouped work quickly
- Spotting gaps in captured context
- Reviewing the map without the full narrative layout

## Table view

Use table view when the work becomes administrative or analytical.

Best for:

- Sorting by status, priority, impact, or effort
- Finding missing owners or stale items
- Cleaning up larger journeys
- Auditing metrics and metadata

## Impact / effort matrix

Use the matrix when the question becomes "what should we do first?"

It helps you compare candidate work by likely value and expected effort without exporting the data somewhere else.

## Prioritization habits that work well

Ask these questions during review:

1. Which customer problem is this about?
2. How often does it happen?
3. How serious is the impact?
4. What evidence supports it?
5. Is the owner clear?
6. Is the next action clear?

## Use filters before you debate

If a journey is large, narrow the view first.

Filter by:

- Group
- Status
- Owner
- Priority
- Impact
- Effort

That keeps prioritization grounded in the exact slice of work the team is discussing.
