Automation templates give your team a named, proven starting point so you can launch a useful workflow in minutes instead of designing one from scratch.

Each template already has a clear trigger pattern, action flow, and AI prompt shape. You still choose the workspace scope, destination, and wording that fit your team.

## How to use a template well

When you choose a template, Custory creates an automation draft.

That draft still needs your decisions:

- which workspace integration to use
- which journey or journeys to scope
- which channel, repo, project, or team to target
- whether the AI instructions need adjustment

Think of a template as a strong starting point, not a finished workflow.

## How to choose the right first template

Start with the smallest workflow that closes a real gap.

Good first choices:

- `Weekly journey pulse` if the problem is visibility
- `Priority focus` if the problem is follow-through
- `GitHub merged PR journey refresh` if the problem is journey drift after shipping
- `PostHog event drift` if the problem is analytics staying disconnected from prioritization

## What to customize after selecting one

Always review:

- scope
- trigger cadence or threshold
- channel or task destination
- journey update instructions
- AI-written wording

Templates are strong defaults, but your team still needs to decide what "useful enough to trust" looks like.

## Template setup mistakes to avoid

<AccordionGroup>
  <Accordion title="Installing too many templates at once">
    Start with one or two workflows the team will actually pay attention to. A smaller automation set is easier to validate, tune, and keep trusted over time.
  </Accordion>
  <Accordion title="Leaving destination fields unset">
    A draft is not useful until the channel, repo, team, or project is configured correctly. Finish the routing details before you treat the template as operational.
  </Accordion>
  <Accordion title="Automating before the journey data is trustworthy">
    Automations amplify the quality of the underlying context. If statuses, owners, or priorities are stale, clean that up first so the template does not spread weak signals.
  </Accordion>
</AccordionGroup>
