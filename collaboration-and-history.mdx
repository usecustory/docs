Custory is built for shared work, not solo note-taking.

## Live collaboration

Inside the journey editor, teams can see:

- Presence avatars
- Active editing cues
- Cursor activity where relevant

This reduces uncertainty when multiple people are in the same journey at once.

## Comments on items

Comments live directly on items.

Use them to:

- Ask follow-up questions
- Clarify evidence
- Record decisions
- Keep discussion attached to the real customer issue

## AI and collaboration

Because Custory AI acts as a workspace participant, AI-assisted work can stay part of the same collaboration flow instead of becoming a separate untracked activity.

## Version history

Version history gives you a snapshot timeline for a journey.

Use it to:

- Review earlier states
- Preview a snapshot before restoring it
- Label important versions
- Restore a prior state when needed

Snapshots can come from manual saves, milestones, session boundaries, and auto-saves.

## When to use version history

Version history is most useful when:

- A major workshop changed the map significantly
- Someone needs to inspect what changed over time
- A restore is safer than manually rebuilding a prior structure
