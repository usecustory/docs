## What the editor helps you do

This page explains the editor as one connected work surface.

It covers:

- journey view
- table view
- impact versus effort
- [search and filters](/search-and-filters)
- [collaboration](/real-time-collaboration)
- [version history](/version-history)

Different kinds of work need different views.

Your team should not have to choose between the customer story and the operational details. The editor lets you move between both.

## The main views

Custory gives you four ways to work in the same journey:

- **Journey view** for the narrative flow across stages and steps
- **Table view** for audits, cleanup, search, sorting, and property review
- **Impact versus effort** for prioritization

These views work on the same underlying journey. You are not creating separate copies of the work.

## Journey view

Use journey view when you need to understand the customer experience in order.

It is best for:

- reviewing the end-to-end path
- explaining the flow to teammates
- spotting where friction builds across stages
- running planning sessions or workshops

Start with the customer path, then inspect the items attached to the moments that matter most.

## Table view

Use table view when you need a more operational scan of the journey.

It is useful for:

- sorting by priority, owner, or status
- finding stale items
- cleaning up incomplete fields
- auditing what has changed recently

Table view is often the fastest way to maintain a mature map.

## Impact versus effort

Use the impact-versus-effort view when the team is deciding what to work on next.

This is best for:

- comparing candidate opportunities
- discussing tradeoffs
- separating quick wins from expensive bets

The goal is not to replace judgment. It is to make prioritization easier to see and discuss.

Read [Prioritize with customer context](/prioritize-with-customer-context) if you want the full workflow for turning evidence into ranked opportunities and linked solutions.

## Search and filters

Use search and filters to narrow large maps quickly.

Custory supports:

- exact-text and semantic search
- filters by item group
- filters by status
- filters by priority, impact, and effort
- filters by owners
- filters by stage and step context

See [Search and filters](/search-and-filters) for the deeper workflow, or [Editor controls](/editor-controls) for the control-level details.

## Property visibility

You can control which properties are shown while you work.

That is useful when a journey has many fields but the current review only needs a smaller surface, such as owners and status or impact and effort.

## Saved working preferences

Custory remembers the last editor setup you used, including view and visibility preferences, so repeated work sessions start closer to the state you expect.

## Collaboration and history

The editor supports live collaboration with:

- presence avatars
- cursor and editing cues
- shared comments on items
- immediate visibility into ongoing work

Use [Real-time collaboration](/real-time-collaboration) when you want to understand the live editing signals, and use [Version history](/version-history) when you need to inspect earlier snapshots, label important states, or restore a prior version safely.

## AI in the editor

[AI as a workspace member](/ai-workspace-member) is available directly where the work happens.

Use it to:

- draft or improve content
- review the current journey
- summarize evidence
- help create follow-up work
- work against connected [integrations](/integrations)

## Editing habits that create friction

## What a healthy editing workflow looks like

A healthy editing workflow lets the team:

- review the customer story clearly
- find important items quickly
- keep fields and ownership current
- prioritize from real evidence
