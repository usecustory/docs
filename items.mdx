Items are the operational layer of a journey.

They hold the customer signal, the team discussion, and the follow-up context that turns a map into something your team can act on.

## Item groups

Custory supports five main item groups.

### Touchpoints

Use touchpoints for customer-facing moments, surfaces, or interactions.

Examples:

- Pricing page
- Signup form
- Support chat
- Billing page

### Insights

Use insights for things your team learned.

Examples:

- Users understand the value but not the setup
- Teams invite others only after first value
- Customers keep asking the same billing question

### Opportunities

Use opportunities for problems or improvement areas worth acting on.

Examples:

- Reduce setup friction for non-technical users
- Improve trial conversion before checkout
- Shorten time to first useful result

### Solutions

Use solutions for candidate fixes or committed improvements.

Examples:

- Add a guided setup flow
- Rewrite the empty state
- Change the billing copy

### Metrics

Use metrics for numbers you need to watch over time.

Examples:

- Activation rate
- Trial-to-paid conversion
- Refund volume
- Time to first value

## Item details

Open an item to work with richer context.

Item details support:

- Files and attachments
- Screenshots and image galleries
- External links with titles and descriptions
- Comments and discussion
- Linked items
- Linked external tasks
- Item history
- Pending AI-related activity where relevant

## Item properties

Depending on the item type, you can use properties such as:

- Status
- Owners
- Priority
- Impact
- Effort
- Confidence
- Severity
- Target date
- Metric direction and values

These properties are what make filtering, prioritization, automations, and notifications useful later.

## Linked tasks and sync

Items can stay connected to work in external tools such as GitHub, Jira, Linear, and Notion.

Custory keeps cached task titles and statuses more current so implementation progress stays visible in the journey after the task is created.

## Comments and collaboration

Use comments when the team needs to ask questions, clarify assumptions, or record why something changed.

Keeping comments on the item prevents the reasoning from disappearing into chat.
