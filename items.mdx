Items are where the real work lives in Custory.

Stages and steps describe the journey structure. Items hold the context that makes that structure useful:

- Evidence
- Customer learning
- Problems worth solving
- Candidate or shipped responses
- Metrics that keep the work honest

## Why items matter

A journey without items is usually just a diagram.

A journey with strong items becomes a working system the team can use for:

- Prioritization
- Delivery handoff
- Cross-functional review
- AI-assisted workflows
- Automation

## The five item groups

Custory supports five core item groups.

### Touchpoints

Customer-facing moments, surfaces, or interactions.

Use them to describe where the experience happens.

Examples:

- Pricing page
- Signup form
- Invite email
- Support chat

See [Touchpoints](/touchpoints).

### Insights

Learnings about the customer experience.

Use them to capture what the team believes, observed, or validated.

Examples:

- Users understand the promise but not the setup
- Buyers care more about rollout clarity than feature depth
- Failed payment emails create avoidable confusion

See [Insights](/insights).

### Opportunities

Problems or leverage points worth acting on.

Use them when the team is ready to frame where intervention could matter.

Examples:

- Reduce setup friction for non-technical admins
- Shorten time to first team value
- Improve billing recovery confidence

See [Opportunities](/opportunities).

### Solutions

Candidate or shipped responses to an opportunity.

Use them for ideas, experiments, features, process changes, content, or service interventions.

Examples:

- Add a guided setup checklist
- Rewrite failed-payment copy
- Create a founder onboarding playbook

See [Solutions](/solutions).

### Metrics

The numbers that tell you whether the problem matters and whether the response worked.

Examples:

- Activation rate
- Median time to first integration
- Trial-to-paid conversion
- Refund volume

See [Metrics](/metrics).

## How items work together

The item model is deliberately connected.

The most common chain is:

- A touchpoint reveals an insight
- An insight leads to an opportunity
- An opportunity is addressed by a solution
- An opportunity or solution is judged by a metric

See [Linking items](/item-linking).

## Item properties

Each item group supports a different set of fields so the metadata stays relevant to the job:

- Touchpoints: channel, actor
- Insights: type, confidence, severity, affected segment
- Opportunities: owner, impact, effort, priority, confidence
- Solutions: owner, solution type, impact, effort, priority, target date
- Metrics: owner, metric type, unit, baseline, current, target, direction

See [Item fields and properties](/item-fields).

## Rich item details

Open an item when you need the full working context.

Item details can include:

- Description
- Files and attachments
- Screenshots and image gallery
- External links with titles and descriptions
- Comments
- Linked items
- Linked external tasks
- Item history
- Pending AI-related activity

See [Rich item details](/rich-item-details).

## How smaller teams should use items

For founder-led and product-led teams, the right goal is not "capture everything."

The right goal is:

- Capture the evidence that changes decisions
- Add the fields that support prioritization and follow-through
- Link the work so the reasoning stays visible

If the item does not help the team decide, act, or learn faster, simplify it.
